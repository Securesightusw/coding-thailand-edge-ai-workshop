# 💡 Idea Canvas — Lean Canvas (4 ช่องแรก)

> ทำช่วงท้ายวัน เปลี่ยน "ของที่รันได้วันนี้" → "ไอเดียนวัตกรรมที่ต่อยอด Day 2 ได้"
> เริ่มจาก **ปัญหา** ไม่ใช่ "เรามี sensor อะไร" · ทำ 4 ช่องนี้ให้คมก่อน ที่เหลือไว้ทีหลัง

- **ชื่อ Prototype (ชั่วคราว):** safe zero ระบบตรวจจับภัยอันตรายในรร

---

## 1️⃣ Problem — ปัญหาคืออะไร

เขียน 2–3 ข้อ ที่มีคน "ลำบาก / เสียเวลา / พลาด" จริง:
```
-การทะเลาะวิวาท
-การก่ออาตชยากรรมต่างๆ
-เหตุการณือื่นๆที่ก่อให้เกิดอันตรายต่อทุกคนในรร
```
**ตอนนี้คนแก้ปัญหานี้ยังไง (ของเดิม)?** ยังไม่ดีพอถังจะมีกล้องแต่ยังไม่สามารถจับปัญหาดังกล่าวได้ ถึงแม้จะมียามก็ตามแต่

## 2️⃣ Customer — ใครคือผู้ใช้จริง

เจาะให้ชัด ไม่ใช่ "ทุกคน": นร ผู้ปกครองนักเรียน ครู และโรงเรียน
```
(เช่น ผู้สูงอายุที่อยู่คนเดียว 60–75  ปี / ช่างซ่อมมอเตอร์ในโรงงาน)
```

## 3️⃣ Value (USP) — คุณค่าเด่นในประโยคเดียว

```safe zero ของเราช่วยเพิ่มความปลอดภัยให้ทุกคนในรรดดยเพิ่มจากเดิมมาให้จำแนกและตรวจจับปัญหาให้ดีขึ้นเพราะทุกปํญหานั้นอาจนำไปสู่เหตุอันตรายที่ยิ่งใหญ่ได้
"[ชื่อของเรา] ช่วย [ผู้ใช้] [ทำอะไรได้]
 ต่างจาก [ของเดิม] เพราะ [ข้อได้เปรียบ]"
```
ตัวอย่าง: *"SafeFall ช่วยผู้สูงอายุที่อยู่คนเดียว ขอความช่วยเหลือได้ทันเมื่อล้ม — ต่างจาก smartwatch แพงๆ เพราะใช้อุปกรณ์ราคาถึงได้และไม่ต้องต่อเน็ต"*

## 4️⃣ Solution — แก้ด้วยอะไร

- **Input ที่ใช้:** กล้อง(Modulino ตัวไหน / กล้อง / ไมค์)
- **Class ที่ AI แยก:** การชกต่อย กลั่นแกล้ง อาวุธ เครื่องแต่งกายที่ส่อถึงอันตรายอย่างไอโม่ และอื่นๆเป็นต้น 
- **Output ตอบสนอง:** ส่งข้อมูลไปยังเทเลแกรมของครูฝ่ายปกครองเพื่อระงับเหตุการณ์ได้อย่างทันท่วงที(Pixels / Buzzer / จอ / log)

---

## 🎯 MVP — ฟังก์ชันเดียวที่ต้องสาธิตให้ได้

```
Input → AI Model → UNO Q → Prediction → Output ที่ผู้ใช้เห็น
```

**1 core function ที่จะโชว์ Day 2/3:** ตรวจจับเหตุการณืต่างๆ แล้วส่งข้อมูลไปยังเทเลแกรมของครู เพื่อระงับเหตุการณือย่างรวดเร็ว และแจ้งเตือนเสียงเมื่อมีอาวุธหรือผู้ก่ออาชยากรรม
*(เช่น "ตรวจได้ว่า Safe/Warning แล้วไฟ+เสียงเตือน")*

> คาถา: **Build less, prove more** — ฟังก์ชันเดียวที่ทำงานจริง + เล่าได้ ดีกว่าหลายอย่างที่ทำไม่สุด
